Restaurant Automatic Reservations System

View Source Code
PHP JavaScript MySQL

About Project

On the front page there is a submission form and in the back there is an admin panel where one can edit and delete customer reservations as well as menu items. I used an html template for the front end to save time and just made small ajustments so it can work with my backend, same goes for dashboard ui.

Customers also receive automated emails whenever their reservation is updated. The e-mail holds all current reservation data as well as a unique link over which the customer can make changes to their reservation at any time.

For the backend I decided not to use any frameworks so I would have to build all the functionality myself and thus get a good grasp on how it all works.

functionality

  • Login system

  • E-mail notifications

  • Reservation and Menu changes from the dashboard

  • Unique url for booking editing

  • Automatised Table Reservation

About Project

A Bug Tracker web app is designed to help customers and support team track bugs, issues and problems.

The app allows users to create an account, log in, create and share bug reports with support team members quickly and easily. Customer account can only see submission they themselves have submitted.

On the other hand, only administrator account (customer support account) can create other administrator accounts. These accounts can see all tickets from all the users, including their own.

functionality

  • Login system

  • Password recovery over E-mail

  • Authentication & Authorization

  • Customers can change profile pictures and information

TaxiStern - Taxi Booking System & Website

View Source Code
PHP MySQL Sass JavaScript JQuery Ajax

About Project

I built this complete website from scratch, everything except the dasboard user interface. for the dashboard UI I used a Graindashboard UI kit, same one I used for Restaurant Dashboard.

A customer can book a drive to and from Vienna Airport. The reservation triggers E-mail notifications to the customer and administrator which holds all important booking information.

In the dashboard, besides booking information, there are also tools an administrator can use to retrieve certain information from a given time frame. Information such as how much income the driver has made, percentage of taken/decliend drives for a given driver or bookings of a certain customer.

I am unable to give source code of this project at the moment due to security concerns as it is a commercial project that is still in use.

functionality

  • Authentication & Authorization

  • Driver and customer notifications over email

  • Orders coming from homepage or entered through admin panel

  • Simple and intuitive functionality that ensures fast operation
