About Project

On the front page there is a submission form and in the back there is an admin panel where one can edit and delete customer reservations. I used a html template for the front end to save time. I just made small ajustments so it can work with my backend.

Customers also receive automated emails whenever their reservation is updated. The e-mail holds all current reservation data as well as a unique link over which the customer can make changes to their reservation at any time.

For the backend I decided not to use any frameworks so I would have to build all the functionality myself and thus get a good grasp on how it all works.

functionality

  • Login system

  • password hashing

  • Authentification

  • E-mail notifications

  • Unique url for booking editing

  • Automatised Table Reservation
